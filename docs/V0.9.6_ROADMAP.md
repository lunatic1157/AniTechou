# AniTechou v0.9.6 Roadmap

v0.9.6 建议聚焦于「桌宠对话闭环 + AI persona 分层 + 发布流程自动化」。v0.9.5 已经把夜子桌宠作为可见入口放到桌面上，下一版应让它真正参与应用交互，而不是只做展示和本地短回复。

## 目标

1. 让桌宠具备可用的 AI 对话入口。
2. 让角色人设可配置、可切换、可扩展。
3. 继续降低 AI 助手逻辑和主窗口代码的耦合。
4. 建立更稳定的发布前检查和打包流程。

## 版本内优先级

### P0: 桌宠 AI 对话闭环

- 在桌宠对话框中接入现有 `AIService`。
- 区分三类输入：
  - 桌宠闲聊：返回短句，显示在桌宠气泡中。
  - 应用命令：转交给主 AI 助手或现有意图分发逻辑。
  - 长回答请求：在主 AI 助手面板展示完整回答，桌宠只显示摘要或提示。
- 增加发送中、失败、取消和重试状态。
- 避免桌宠气泡承载长篇回复，桌面层只保留短反馈。

### P0: Persona 分层

- 将 AI prompt 拆成两层：
  - 能力层：AniTechou 助手能做什么，例如搜索、推荐、补全、整理笔记。
  - 角色层：角色说话方式、人设边界、禁用方向、短句风格。
- 当前默认角色为夜子，保留丹特丽安资料作为后续角色包参考。
- `AIService` 不直接写死角色人设，由配置或角色 manifest 提供 persona prompt。

### P1: 角色包 manifest

- 角色包建议包含：
  - `id`
  - `displayName`
  - `description`
  - `assetRoot`
  - `states`
  - `localLines`
  - `personaPrompt`
  - `safetyNotes`
  - `fallbackReplies`
- 增加 manifest 校验，检查必需状态资源是否存在、PNG 是否透明、尺寸是否符合预期。
- 为以后新增角色保留目录迁移方案，逐步从 `Assets/Pets/Dalian` 过渡到 `Assets/Pets/Yoruko` 或 `Assets/Pets/<RoleId>`。

### P1: 桌宠功能完善

- 对话框增加历史消息区或最近 3 条短消息。
- 增加“打开主 AI 助手”按钮，便于长对话转到主界面。
- 增加桌宠缩放设置。
- 增加“回到默认位置”。
- 增加状态显示或调试模式开关，便于检查 idle/thinking/pleased/annoyed/moving/sleeping。

### P2: AI 助手架构整理

- 从 `MainWindow.xaml.cs` 拆出 AI 命令控制器。
- 将推荐、补全、聊天、桌宠对话的入口分开，但共用底层 `AIService` 和配置。
- 为 prompt 构建增加单元测试，重点覆盖 persona 注入、角色切换和不污染结构化 JSON 输出。

### P2: 发布自动化

- 增加发布脚本，按顺序执行：
  - `dotnet build -c Release`
  - `dotnet test Tests\AniTechou.Tests.csproj -c Release`
  - `dotnet publish AniTechou.csproj -c Release -r win-x64 --self-contained true -o publish\win-x64-self-contained`
  - Inno Setup 打包
  - 计算安装包 SHA256
  - 生成 GitHub Release 草稿摘要
- 发布前检查版本号一致：
  - `AniTechou.csproj`
  - `Installer/AniTechou.iss`
  - Git tag
  - GitHub Release 标题和资源文件名

## 非目标

- v0.9.6 不建议一次性重做整个 AI 助手 UI。
- 不建议把所有角色资源都接入在线生成流程，先把本地角色包格式稳定下来。
- 不建议让桌宠直接执行高风险写操作；涉及数据修改时仍应回到主窗口确认。

## 验收标准

- 打开桌宠对话框后，可以完成一次真实 AI 闲聊并显示短回复。
- 切换 persona 后，AI 回复语气变化，但搜索、推荐、补全等结构化能力不退化。
- 角色 manifest 缺资源时能给出明确错误，而不是显示空白桌宠。
- Release 构建、测试、publish 和安装器打包可以通过一条脚本完成。
- 文档包含面向用户的更新说明、开发者变更说明和下一步规划。
